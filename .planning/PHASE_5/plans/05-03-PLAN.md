# Plan 05-03: Smart Hint UI & Walkthrough System

## Goal
Implement the progressive disclosure Hint UI with logical explanations, animations, and "Pattern Solved!" feedback.

## Tasks
1. **Hint State Management**:
   - Add `activeHint: Hint | null` to `useGameStore` to track current hint.
   - Add `isExplainingMore: boolean` to track the walkthrough state.
2. **Hint Component**:
   - Create `src/components/ui/HintExplanation.tsx` for the progressive disclosure UI.
   - Implement "Hint" button -> "Label" -> "Explain More" -> "Action" flow.
3. **Logic Visuals & Animations**:
   - Use the "Logic Gold" (or Purple) color for hint-related cell and note highlights.
   - Implement animated candidate strike-throughs using CSS or Framer Motion.
4. **Pattern Solving Feedback**:
   - Add logic to detect if a cell is solved *after* a hint but before being revealed.
   - Create a subtle "Pattern Solved!" toast or animation.
5. **Fix Notes Fallback**:
   - Implement the "Fix Notes" option when no logical pattern is found.
6. **Verification**:
   - Trigger a "Naked Pair" hint, follow the "Explain More" walkthrough, and verify the correct cells/notes are highlighted.
   - Confirm that the strike-through animation plays as expected.

## Technical Details
- **Hint Type**:
  ```typescript
  interface Hint {
    strategy: string;
    description: string;
    targetIndices: number[];
    reasonIndices: number[]; // Cells that "prove" the logic
    value?: number;
    eliminates?: { index: number; value: number }[];
  }
  ```
- **Styling**: Use consistent logic-specific theme colors.
